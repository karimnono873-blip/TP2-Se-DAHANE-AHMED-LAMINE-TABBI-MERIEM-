1. Jumeau Numérique & Code C

Objectif : Lire une tension entre 0V et 3.3V provenant d'un potentiomètre sur PA0. La LED sur PA5 s'allume si la valeur dépasse 50% de la pleine échelle (VREF/2).

Potentiomètre (PA0)
ADC Brute : 0
Tension : 0.00 V

LED (PA5)

Code de lecture (Polling)

Le microcontrôleur interroge séquentiellement le convertisseur analogique-numérique (ADC1) configuré sur une résolution de 12 bits.

// Boucle infinie - main.c
while (1) 
{ 
    // Lance l'échantillonnage matériel
    HAL_ADC_Start(&hadc1); 
 
    // Attente bloquante (max 100ms) de la fin de conversion SAR
    HAL_ADC_PollForConversion(&hadc1, 100); 
 
    // Récupération de la valeur numérique dans le registre de données
    adcResult = HAL_ADC_GetValue(&hadc1); 
 
    // Arrêt de l'ADC pour économiser l'énergie de la carte
    HAL_ADC_Stop(&hadc1); 
 
    // Logique décisionnelle (Seuil à ~1.65V)
    if (adcResult < 2048) 
    { 
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); 
    } 
    else 
    { 
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET); 
    } 
 
    HAL_Delay(10); 
}

2. Post-Lab : Analyse Technique Approfondie

L'ADC du STM32C0 est configuré sur 12 bits. Quelle est la valeur numérique maximale ? Si on passait en 8 bits, quelle serait cette valeur ?

La résolution d'un ADC détermine le nombre de pas discrets qu'il peut utiliser pour représenter un signal analogique. La formule générale du nombre de valeurs est 2n, où n est le nombre de bits.

Pour un ADC 12 bits, on a 212 = 4096 niveaux possibles. Puisque l'on commence à compter à partir de zéro, la valeur numérique maximale retournée par le registre de données sera :

Valeur Max (12 bits) = 212 - 1 = 4095

Le "Quantum" (ou LSB - Least Significant Bit) pour une tension de référence de 3.3V est très précis : q = 3.3V / 4095 ≈ 0.80 mV. Le microcontrôleur détecte donc des variations de 0.8 millivolts.

Si l'ADC était configuré avec une résolution de 8 bits pour économiser de la mémoire (Direct Memory Access plus rapide par exemple) :

Valeur Max (8 bits) = 28 - 1 = 255

La précision chuterait considérablement : q = 3.3V / 255 ≈ 12.9 mV.

À quoi sert la fonction HAL_ADC_PollForConversion() ? Que se passerait-il si on l'omettait ?

L'ADC des STM32 utilise une architecture matérielle appelée SAR (Successive Approximation Register). La conversion du signal analogique capturé par l'échantillonneur-bloqueur (Sample & Hold) vers un format binaire n'est pas instantanée. Elle requiert plusieurs cycles d'horloge de l'ADC (environ 12.5 cycles minimum).

La fonction HAL_ADC_PollForConversion(&hadc1, 100) est une fonction bloquante (mode Polling). Elle met le processeur principal en boucle d'attente jusqu'à ce que le bit EOC (End Of Conversion) soit levé dans le registre de statut matériel, signalant que la donnée est prête. Le paramètre 100 est un Timeout de sécurité (en millisecondes) pour éviter que le système ne plante si l'ADC subit une défaillance matérielle.

Conséquence de son omission : Le processeur exécuterait immédiatement HAL_ADC_GetValue() avant que la conversion SAR ne soit terminée. Il lirait alors le registre de données au mauvais moment, récupérant soit un "0", soit du "bruit" (garbage data), soit l'ancienne valeur de la conversion précédente.

Pourquoi avons-nous choisi la valeur 2048 comme seuil dans le test if ? À quelle tension précise en Volts cela correspond-il ?

La valeur 2048 n'a pas été choisie au hasard : elle représente exactement la moitié de l'échelle d'un convertisseur 12 bits (4096 / 2 = 2048). Cela correspond mécaniquement au point central (50%) de la course de notre potentiomètre, offrant une interface homme-machine logique.

D'un point de vue électrique, pour calculer la tension analogique exacte correspondante, on utilise la formule de conversion inverse :

Tension (V) = (Valeur ADC × VREF+) / (2n - 1)

Tension = (2048 × 3.3V) / 4095 ≈ 1.650 Volts

Le seuil de basculement de la LED est donc précisément réglé à 1.65V.

Pourquoi appelle-t-on HAL_ADC_Stop() à la fin de chaque lecture ? Quel impact sur la consommation d'énergie ?

L'ADC est un périphérique analogique. Contrairement aux périphériques purement numériques, il intègre des composants physiques gourmands (réseau de condensateurs commutés, amplificateurs opérationnels, comparateurs de tension).

Lorsqu'on appelle HAL_ADC_Start(), ces circuits sont mis sous tension et commencent à consommer un "courant statique" continu, même lorsqu'ils ne sont pas activement en train de convertir. Appeler HAL_ADC_Stop() permet de couper l'horloge du périphérique et d'éteindre l'étage analogique interne.

Impact énergétique : Dans le cadre d'un système embarqué fonctionnant sur batterie (IoT, capteurs portables), laisser l'ADC allumé en permanence diviserait la durée de vie de la batterie de manière drastique. L'éteindre entre deux mesures espacées (Duty Cycling) est une des règles d'or du Low-Power Design en ingénierie électronique.

Comment éviter le clignotement instable de la LED au milieu du potentiomètre (concept d'hystérésis) ?

Avec un seuil unique strict à 2048, le système souffre d'instabilité. Si le potentiomètre est physiquement placé à 1.65V, le bruit électromagnétique naturel (EMI) et les imperfections mécaniques de la piste en carbone vont faire osciller la lecture de l'ADC (ex: 2047, 2049, 2046, 2048...). Le processeur va donc allumer et éteindre la LED à une vitesse fulgurante (phénomène de Chattering).

La solution professionnelle est de créer une Hystérésis logicielle (Bascule de Schmitt) en définissant deux seuils distincts espacés par une "zone morte" (Deadband) qui absorbera le bruit :

// Implémentation d'une hystérésis de 100 unités
if (adcResult > 2100) { 
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET);   // Seuil d'allumage haut
} 
else if (adcResult < 2000) { 
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); // Seuil d'extinction bas
}
// Entre 2000 et 2100, on ne fait rien, la LED garde son état précédent.
Si on remplace le potentiomètre par un capteur de température LM35 (Sensibilité : 10mV / °C), quelle est la formule C pour obtenir la température exacte ?

Le capteur LM35 est un capteur linéaire. Il délivre 0V à 0°C, et sa tension de sortie augmente de 10 millivolts pour chaque degré Celsius supplémentaire (ex: 250mV = 25°C).

La conversion nécessite deux étapes mathématiques que l'on peut simplifier dans le code :
Étape 1 : Calculer la tension lue en millivolts (mV).
VmV = (adcResult × 3300.0) / 4095.0

Étape 2 : Diviser par la sensibilité (10 mV/°C) pour isoler les degrés.
Temp = VmV / 10.0

En factorisant cette équation pour optimiser le temps de calcul du processeur ARM (3300 divisé par 10 devient 330), on obtient le code C suivant :

float temperature = (adcResult * 330.0f) / 4095.0f;

*Note technique : On utilise des nombres flottants avec le suffixe 'f' pour forcer le compilateur à utiliser l'unité de calcul flottante (FPU) simple précision du microcontrôleur, évitant ainsi la troncature liée aux divisions d'entiers.*
